# API Authentication

The Nimbus BCI Engine uses API keys to authenticate requests. All API endpoints require authentication to ensure secure access to your BCI models and data.

## Getting Your API Key

To get started with the Nimbus API, you'll need to obtain an API key:

<Card title="Request API Access" icon="envelope" href="mailto:hello@nimbusbci.com">
  Contact us at **hello@nimbusbci.com** to request your API key and discuss your BCI use case.
</Card>

<Note>
API keys are currently issued on a case-by-case basis. Please include details about your intended use case, expected volume, and timeline in your request.
</Note>

## Authentication in Julia SDK

The NimbusSDK.jl package handles authentication automatically. You authenticate once when initializing the SDK:

```julia
using NimbusSDK

# Authenticate with your API key
session = NimbusSDK.authenticate("your-api-key-here")

# SDK is now authenticated and ready to use
# Credentials are cached locally for 30 days
```

## API Endpoints

The Nimbus API provides authentication, license management, and analytics endpoints (inference is handled client-side via the Julia SDK).

### POST /v1/auth/validate

Validate an API key and return user permissions.

**Request:**
```json
{
  "api_key": "nbci_live_..."
}
```

**Response:**
```json
{
  "valid": true,
  "user_id": "...",
  "license_type": "research",
  "features": ["batch_inference", "streaming"],
  "quota": {
    "monthly_limit": 50000,
    "used": 1234,
    "remaining": 48766,
    "resets_at": "2025-11-01T00:00:00Z"
  }
}
```

### POST /v1/auth/refresh

Refresh API key credentials and quota information.

### POST /v1/auth/revoke

Revoke an API key (makes it inactive).

### Authentication Headers

| Header | Value | Required |
|--------|-------|----------|
| `Content-Type` | `application/json` | ✅ Yes |
| `api_key` | Your API key | ✅ Yes |

## Security Best Practices

### Protect Your API Key

<Warning>
**Never expose your API key in client-side code, public repositories, or logs.** API keys should only be used in secure server environments.
</Warning>

**✅ Do:**
- Store API keys in environment variables
- Use secure key management systems
- Rotate keys regularly
- Monitor API usage for anomalies

**❌ Don't:**
- Hardcode keys in source code
- Share keys via email or chat
- Use the same key across multiple environments
- Commit keys to version control

### Environment Variables

Store your API key securely:

```bash
# Environment variable
export NIMBUS_API_KEY="nbci_live_..."
```

```julia
using NimbusSDK

# In Julia, store API key in ~/.nimbus/api_key (auto-created)
# Or set environment variable
key = ENV["NIMBUS_API_KEY"]

# Authenticate
session = NimbusSDK.authenticate(key)
```

## Rate Limits

API requests are subject to rate limits to ensure fair usage and system stability:

| License Type | Monthly Requests | Features |
|------|-------------------|----------------|
| **Free** | 50,000 | Batch inference, basic models |
| **Research** | 100,000 | Batch + streaming, advanced models, custom training |
| **Commercial** | 500,000 | All features, priority support, custom models |
| **Enterprise** | Unlimited | All features, dedicated support, custom deployments |

### Rate Limit Headers

Response headers indicate your current rate limit status:

```http
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 299
X-RateLimit-Reset: 1640995200
X-RateLimit-Window: 60
```

### Handling Rate Limits

When you exceed rate limits, the API returns a `429 Too Many Requests` response:

```json
{
  "error": {
    "code": "rate_limit_exceeded",
    "message": "Rate limit exceeded. Try again in 60 seconds.",
    "retry_after": 60
  }
}
```

**Best practices for handling rate limits:**
- Implement exponential backoff
- Cache credentials locally (SDK does this automatically)
- Use Julia SDK's streaming mode for real-time data (client-side)
- Contact us to upgrade your license for higher limits

## Error Responses

Authentication errors return specific status codes and messages:

### 401 Unauthorized

```json
{
  "error": {
    "code": "unauthorized",
    "message": "Invalid or missing API key"
  }
}
```

**Common causes:**
- Missing `Authorization` header
- Invalid API key format
- Expired or revoked API key

### 403 Forbidden

```json
{
  "error": {
    "code": "forbidden",
    "message": "API key does not have permission for this resource"
  }
}
```

**Common causes:**
- API key lacks required permissions
- Accessing restricted endpoints
- Account suspension or limitations

## API Key Management

### Key Rotation

For security, we recommend rotating API keys regularly:

1. **Request a new key** by contacting hello@nimbusbci.com
2. **Update your applications** with the new key
3. **Test thoroughly** in staging environment
4. **Deploy to production** during maintenance window
5. **Revoke the old key** after successful deployment

### Multiple Environments

Use different API keys for different environments:

```bash
# Development
NIMBUS_API_KEY_DEV=dev_key_here

# Staging  
NIMBUS_API_KEY_STAGING=staging_key_here

# Production
NIMBUS_API_KEY_PROD=prod_key_here
```
