# API Endpoints

The Nimbus API provides **authentication, licensing, analytics, and model registry** services. **Inference is handled client-side** via the NimbusSDK.jl Julia package, not through API endpoints.

## Base URL

```
https://api.nimbusbci.com/v1
```

## Architecture

<Note>
**Important**: The Nimbus API does **not** provide inference endpoints. All BCI inference happens **client-side** using the Julia SDK (NimbusSDK.jl) for maximum performance and privacy.
</Note>

The API provides:
- **Authentication** - API key validation and authorization
- **License Management** - Quota tracking, feature gates
- **Analytics** - Usage logging and metrics
- **Model Registry** - Model catalog and metadata

## Authentication Endpoints

### POST /v1/auth/validate

Validate an API key and return user permissions.

**Request:**
```bash
curl -X POST "https://api.nimbusbci.com/v1/auth/validate" \
  -H "Content-Type: application/json" \
  -d '{
    "api_key": "nbci_live_..."
  }'
```

**Response:**
```json
{
  "valid": true,
  "user_id": "...",
  "license_type": "research",
  "features": ["batch_inference", "streaming", "custom_training"],
  "quota": {
    "monthly_limit": 100000,
    "used": 1234,
    "remaining": 98766,
    "resets_at": "2025-02-01T00:00:00Z"
  },
  "expires_at": "2025-12-31T23:59:59Z"
}
```

### POST /v1/auth/refresh

Refresh API key credentials and update quota information.

**Request:**
```bash
curl -X POST "https://api.nimbusbci.com/v1/auth/refresh" \
  -H "Content-Type: application/json" \
  -d '{"api_key": "nbci_live_..."}'
```

### POST /v1/auth/revoke

Revoke an API key (makes it inactive).

**Request:**
```bash
curl -X POST "https://api.nimbusbci.com/v1/auth/revoke" \
  -H "Content-Type: application/json" \
  -d '{"api_key": "nbci_live_..."}'
```

## Analytics Endpoints

### POST /v1/analytics/log

Log inference events and usage metrics (called automatically by Julia SDK).

**Request:**
```json
{
  "api_key": "nbci_live_...",
  "event_type": "inference",
  "model_name": "motor_imagery_4class_v1",
  "inference_type": "batch",
  "duration_ms": 245,
  "metadata": {
    "n_trials": 20,
    "mean_confidence": 0.85
  }
}
```

<Note>
The Julia SDK handles analytics logging automatically. You typically don't need to call this endpoint directly.
</Note>

## Model Registry

### GET /v1/models/list

List available models based on your license.

**Query Parameters:**
- `api_key` (required)
- `paradigm` (optional) - Filter by BCI paradigm (`motor_imagery`, `p300`, `ssvep`)
- `type` (optional) - Filter by model type (`RxLDA`, `RxGMM`)

**Request:**
```bash
curl -X GET "https://api.nimbusbci.com/v1/models/list?api_key=nbci_live_...&paradigm=motor_imagery" \
  -H "Content-Type: application/json"
```

**Response:**
```json
{
  "models": [
    {
      "name": "motor_imagery_4class_v1",
      "version": "1.0.0",
      "type": "RxLDA",
      "paradigm": "motor_imagery",
      "n_features": 16,
      "n_classes": 4,
      "requires_license": "free",
      "download_url": "https://models.nimbusbci.com/motor_imagery_4class_v1.jld2",
      "checksum": "...",
      "file_size_mb": 0.5
    }
  ],
  "total": 1
}
```

## How Inference Works

Since the API does **not** provide inference endpoints, here's how BCI inference works with Nimbus:

### 1. Client-Side Inference (Julia SDK)

```julia
using NimbusSDK

# Authenticate with API (validates license)
NimbusSDK.authenticate("nbci_live_...")

# Load model (may download from API registry)
model = load_model(RxLDAModel, "motor_imagery_4class_v1")

# Run inference locally (fast, private)
data = BCIData(features, metadata, labels)
results = predict_batch(model, data)  # Client-side!
```

### Benefits of Client-Side Inference

1. **Ultra-low latency** - No network round-trip
2. **Privacy** - EEG data never leaves your machine
3. **Performance** - Sub-10ms inference locally
4. **Offline capable** - Works without internet after initial authentication
5. **Scalability** - No server load for inference

### 2. API Provides Authentication Only

```julia
# Authentication happens once
session = NimbusSDK.authenticate("api-key")

# Credentials cached locally for 30 days
# Subsequent inference doesn't call API
results = predict_batch(model, data)  # No API call!
```

## License Tiers

API validates your license and grants access to appropriate features:

| Tier | Monthly Quota | Features |
|------|---------------|----------|
| **Free** | 50,000 | Basic models, batch inference |
| **Research** | 100,000 | All models, batch + streaming, custom training |
| **Commercial** | 500,000 | All features, priority support, custom models |
| **Enterprise** | Unlimited | All features, dedicated support |

## Error Responses

### 401 Unauthorized

```json
{
  "error": "Invalid or expired API key",
  "code": "UNAUTHORIZED"
}
```

### 403 Forbidden

```json
{
  "error": "License insufficient for requested feature",
  "code": "FORBIDDEN",
  "required_license": "research",
  "current_license": "free"
}
```

### 429 Rate Limit Exceeded

```json
{
  "error": "Quota exceeded. Reset at start of next month",
  "code": "QUOTA_EXCEEDED",
  "retry_after": "2025-02-01T00:00:00Z"
}
```

## Example: Complete Workflow

```julia
using NimbusSDK
using HTTP, JSON

# 1. Authenticate via API
NimbusSDK.authenticate("nbci_live_...")

# 2. List available models from API
models_url = "https://api.nimbusbci.com/v1/models/list"
response = HTTP.get("$models_url?api_key=nbci_live_...")
models = JSON.parse(String(response.body))

println("Available models:")
for m in models["models"]
    println("  $(m["name"]) - $(m["type"])")
end

# 3. Load and use model locally (no API calls)
model = load_model(RxLDAModel, "motor_imagery_4class_v1")
results = predict_batch(model, data)

# 4. Analytics logged automatically by SDK
# (runs in background, doesn't block inference)
```
